디자인 없이 개발하게 된 계기

  • 이를 바탕으로 피그마 없이 프론트엔드 개발자가 직접 UI/UX 설계 및 제안 가능

사업진행부서와 긴밀한 협업

  • 사업팀과 주기적인 화면 설계 회의 진행, 합리적인 화면 구성을 협의하며 개발

  • User 앱과 Admin 앱을 별도로 관리할 경우 가이드 문서를 각각 작성해야 하는 비효율 발생
  • 공통 컴포넌트, 타입, 유틸리티를 shared 패키지로 통합 관리하여 문서화 부담 최소화
  • 컴포넌트는 공유하되, 배포는 User/Admin 각각 독립적으로 진행
  • 코드 일관성 유지 및 유지보수 효율성 향상

2단계 어드민 권한 구조 설계 및 구현

  • 슈퍼어드민(SystemSidebar): 전시회 프로젝트 CRUD, 관리자 계정 관리, 시스템 전역 설정
  • 프로젝트 어드민(AdminSidebar): 개별 전시회별 참가신청 관리, 신청 상태 변경, 인보이스 발행
  • JWT 기반 권한 검증 및 역할별 메뉴 접근 제어
  • HOC(Auth, AdminAuth) 패턴으로 역할 기반 권한 분기 처리

참가신청 관리 시스템

  • 예비신청/정규신청 탭 분리로 신청 단계별 목록 관리
  • 신청 상태별 유저 권한 관리: 대기 상태 시 정보 수정 가능, 승인 시 수정 불가 및 인보이스 조회 활성화
  • TanStack Query 기반 실시간 데이터 동기화

복잡한 어드민 리스트 UI/UX 개선

  • onChange 기반 실시간 검색 처리 및 필터/페이지네이션 구현
  • 로우 데이터가 길어 한 화면에 모두 표시해야 하는 요구사항 → 가로/세로 스크롤 중첩 시 UI 저하 문제를 스와이프 제스처로 해결
  • 툴팁을 활용한 긴 텍스트 및 상태 정보 가시성 개선
  • 컬럼 수가 많은 테이블에서 핵심 정보 우선 노출 전략 적용

입금 관리 및 인보이스 발행
