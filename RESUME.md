### 1. O2MEET 온라인 행사 플랫폼 개발 및 유지보수

**기간**: 2023.05 ~ 현재
**서비스 홍보 사이트**: https://o2meet.io
**기술 스택**: JSP, jQuery

#### 플랫폼 개요

O2MEET은 온/오프라인 행사를 위한 통합 플랫폼으로, 다음 솔루션을 제공합니다:

| 솔루션 | 설명 |
|--------|------|
| **웹사이트 솔루션** | 행사 홍보 페이지 제작 툴 |
| **비즈매칭 솔루션** | 온/오프라인 비즈니스 미팅 스케줄 관리, 화상 미팅, 설문조사 |
| **전시 패키지** | CDN 영상, 유튜브, 이미지 슬라이드 등 다양한 콘텐츠 전시 |
| **회의 패키지** | 컨퍼런스 생중계, 연사 소개, VOD 재생 |
| **상품 솔루션** | 온라인 티켓 구매 및 QR 코드 문자메시지 발송 |
| **관리자 페이지** | 행사 기획자를 위한 통합 관리 시스템 |

#### 담당 업무

- 온라인 전시관, 컨퍼런스, 티켓 구매 및 등록 서비스 프론트엔드 개발
- 여러 행사에서 공통적으로 사용되는 API 호출 부분 공통화 작업
- 프로젝트별 특성에 맞는 UI/UX 구현
- 복잡한 마이페이지 UI/UX 및 데이터 처리 구현
- 상품 관리 및 결제 시스템 프론트엔드 구현

##### 2) 온라인 전시관 시스템 구현
- 사용자 권한에 따른 메뉴 접근 제어 (전시사 유저 전용 콘텐츠 등록 메뉴)
- 카테고리 API 기반 전시사 리스트 동적 구성
- 다양한 미디어 지원 (이미지, MP4 영상, 유튜브)
- 북마크 기능 구현

##### 3) 카테고리 API 활용 최적화
- 참가목적, 기업 분류 등 데이터 기반 프론트엔드 구현으로 다국어 처리 작업 최소화
- 상품 분류 시스템 설계 및 기획자 제안을 통한 도입

#### 아키텍처 의사결정

##### Atomic Design Pattern 도입
- **재사용성 극대화**: Atoms → Molecules → Organisms → Templates → Pages 단위 구조화
- **일관된 UI 경험**: 동일 디자인 요소 재활용으로 사용자 경험 통일
- **개발 효율성**: 기본 구성 요소 조합으로 복잡한 UI 구현
- **유지보수 용이성**: 컴포넌트 단위 수정으로 전체 시스템 반영
- **협업 효율 향상**: 디자이너-개발자 간 공통 언어 제공

##### Emotion (CSS-in-JS) 채택
- CSS-in-JS 방식으로 JavaScript 내 스타일 정의
- props 기반 동적/조건부 스타일링
- 컴포넌트별 스타일 스코프 격리로 충돌 방지
- Code Splitting 최적화
- TypeScript 우수한 통합성
- SSR 지원으로 Next.js 호환성 확보
- 테마 시스템을 통한 전역 스타일 관리

##### 상태 관리 전략 변경
- **기존**: Redux 기반 설계 → 복잡한 파일 구조 및 보일러플레이트 문제
- **변경**: React Query + Zustand 조합
  - **React Query**: 서버 상태 관리, 캐싱, 로딩/에러 상태 자동 처리
  - **Zustand**: 간결한 문법, 보일러플레이트 최소화, TypeScript 연동 용이
  - 팀원 교육 및 적용 난이도 감소로 마이그레이션 속도 향상

### 3. 전시회 부스 신청 관리 시스템 (Admin) - 진행중

**기간**: 2024.10 ~ 현재 (진행중)
**기술 스택**: Next.js 16, React 19, TypeScript, TanStack Query, Zustand, Tailwind CSS 4, Turborepo
**프로젝트 구조**: Turborepo 모노레포 (User App + Admin App + Shared Packages)

#### 프로젝트 개요

전시회 진행 시 부스를 사전에 신청하는 업체들을 **전시회별로 통합 관리**하는 어드민 시스템입니다. 슈퍼어드민과 프로젝트 어드민의 **2단계 권한 구조**로 설계하여, 여러 전시회를 동시에 운영하면서도 각 전시회별 독립적인 관리가 가능합니다.

#### Admin 시스템 구조

| 권한 | 역할 | 주요 기능 |
|------|------|-----------|
| **슈퍼어드민** | 시스템 전체 관리 | 프로젝트(전시회) CRUD, 관리자 계정 관리, 시스템 설정 |
| **프로젝트 어드민** | 개별 전시회 관리 | 참가신청 관리, 업체 정보 조회, 상태 변경, 인보이스 발급 |

#### 담당 업무 및 역할

- **프론트엔드 전체 설계 및 구현** (Admin App 메인 담당)
- 슈퍼어드민/프로젝트 어드민 권한 분기 UI 설계
- 데이터 테이블 및 상세 페이지 UI/UX 설계
- 공유 컴포넌트 라이브러리 설계 및 구현 (26+ 컴포넌트)

#### 주요 기술적 구현

##### 1) 2단계 어드민 사이드바 시스템
- **SystemSidebar**: 슈퍼어드민 전용 좁은 사이드바 (아이콘 + Tooltip)
- **AdminSidebar**: 프로젝트별 사이드바 (접기/펼치기, 다국어 지원, 로컬스토리지 상태 저장)
- JWT 토큰 기반 권한 검증 및 자동 라우팅

##### 2) 참가신청 관리 시스템
- **예비/정규 신청 탭 분리**: 신청 단계별 목록 관리
- **검색/필터/페이지네이션**: 대량 데이터 효율적 처리
- **상태 관리**: 대기 → 승인 → 반려 등 워크플로우 구현
- TanStack Query 기반 실시간 데이터 동기화

##### 3) 데이터 테이블 UI 직접 설계
- **Sticky 컬럼 구현**: 좌우 스크롤 시에도 번호/업체명 등 핵심 정보 고정
- **고정 너비 + 그림자 효과**: 고정 영역과 스크롤 영역 시각적 구분
- **반응형 전환**: 데스크톱(테이블 뷰) ↔ 모바일(카드 뷰) 자동 전환

##### 4) 상세 페이지 고정 헤더 (직접 제안한 UI)
- **문제 발견**: 운영팀이 여러 탭을 오가며 작업 시 핵심 정보 확인 어려움
- **해결책 제안**: 사업팀에 **상단 고정 헤더** 직접 제안 및 승인
- **구현 내용**:
  - 회사명, 상태, ID, 등록일, 신청일 상시 노출
  - **상태 뱃지 클릭**으로 상세 페이지에서 바로 상태 변경 가능
  - 탭 이동/스크롤해도 핵심 정보 항상 확인 가능

##### 5) 공유 컴포넌트 라이브러리 (packages/shared)
```
Button, Input, Select, Modal, DataTable, ResponsiveTable,
DynamicForm, TabNav, ProgressBar, StatusBadge, Toast, Alert,
ContentCard, UserDetailBottomBar... (26+ 컴포넌트)
```
- Turborepo 모노레포 구조로 User App과 Admin App에서 공유
- TypeScript 기반 Props 타입 정의로 재사용성 극대화

#### 주도적 UI/UX 설계 사례

| 설계 항목 | 내용 |
|-----------|------|
| **Sticky 컬럼 테이블** | 긴 테이블에서 핵심 정보 항상 확인 가능 |
| **실시간 업데이트** | TanStack Query로 페이지 새로고침 없이 데이터 반영 |
| **반응형 테이블** | 화면 크기에 따른 테이블/카드 뷰 자동 전환 |
| **고정 헤더** | 운영팀 업무 흐름 파악 후 **직접 제안**하여 도입 |
| **비개발자 협업** | Before/After 스크린샷으로 명확한 설명, 빠른 피드백 사이클 |

#### 성과

- **운영팀 업무 효율성 향상**: 고정 헤더 도입으로 탭 이동 시에도 핵심 정보 즉시 확인
- **개발 생산성 향상**: 공유 컴포넌트 라이브러리로 신규 기능 개발 속도 향상
- **확장 가능한 구조**: 신규 전시회 추가 시 프로젝트 생성만으로 즉시 운영 가능
